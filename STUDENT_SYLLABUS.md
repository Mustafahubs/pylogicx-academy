## How Every Week Works

No week is a lecture you sit through passively. Every week follows the same rhythm:

```
The Hook       → A real problem you'll actually want to solve
Theory         → The minimum you need to know — explained through the project
Build It       → You write the code, guided step by step
AI Lab         → Structured practice using ChatGPT/Claude as your pair programmer
Assignment     → Your own version to submit and show off
```

**One rule about AI tools:** You're allowed — encouraged — to use them. But if you can't explain every line of AI-generated code in your own words, you haven't learned it yet. We enforce this.

## No Exams. Ever.

Assessment is 100% based on what you produce.

| What | Weight |
|------|--------|
| Weekly working script (submitted each week) | 50% |
| AI Lab participation | 20% |
| Capstone project (Weeks 13–14) | 20% |
| Code walkthrough — explain your own code verbally | 10% |

**The standard:** Code that runs and does what it's supposed to do. It doesn't have to be perfect. It has to work.

## The One Thing That Will Determine Your Success

At the end of every week, you show your project to at least one person outside class.

Not for a grade. Because explaining what you built — even to someone who doesn't code — forces a level of understanding that no test can measure. The students who do this every week finish. The ones who skip it tend not to.
